# 产品需求库 (PRD Hub) — 使用指南

> 跨端产品需求文档库。承接 PM 输入；后端 / 前端 / iOS / Android 等各端有独立的实现方案 wiki，通过链接相互连接。

## 0. 这里放什么 / 不放什么

✅ **放在这个空间**：
- 产品需求文档（PRD）
- 产品规划 / Roadmap / OKR 拆解
- 跨端业务规则、状态机、用户流程
- A/B 实验设计、运营策略
- 数据指标定义（业务侧，非埋点字段实现）

❌ **不放这个空间**：
- 单端实现方案、API 字段定义、DB schema → **去对应端的实现方案 wiki**
- 代码片段、生成的接口产物 → **repo**
- 故障复盘、上线回滚 → **去对应端 wiki 的 ops/**

## 1. 跨空间协作规则（铁律）

**一个需求 = 1 篇 PRD（这里）+ N 篇实现方案（各端 wiki）**

| 角色 | 写哪 | 链给谁 |
|------|------|--------|
| PM | 本空间写 PRD | 在 PRD 文末"实现追踪"区贴各端方案链接 |
| 后端工程师 | 后端 wiki `features/` | 文档头部"对应 PRD"字段链回本空间 |
| 前端工程师 | 前端 wiki `features/` | 同上 |
| iOS / Android | 各自 wiki `features/` | 同上 |

**双向链接是这套体系的命门**——任何一篇文档都要能在 30 秒内跳到上下游。

## 3. PRD 状态约定（标题前缀）

| 前缀 | 含义 |
|------|------|
| `[Draft]` | 草稿，PM 起草中 |
| `[Review]` | 评审中 |
| `[WIP]` | 已排期，开发中 |
| `[Done]` | 已上线 |
| `[Hold]` | 暂停 / 排期外 |
| `[Cancelled]` | 取消 |

示例：`[WIP] 2026-Q2 xxx 功能`、`[Done] 2026-01 xxx 重构`

## 4. 写新 PRD

1. 复制 [`_PRD 模板`](./_PRD%20%E6%A8%A1%E6%9D%BF.md)
2. 标题：`[状态前缀] YYYY-MM 需求名`
3. 完成后在文末"实现追踪"表格里贴各端实现方案链接

## 6. 已上线 PRD 复用建议

老 PRD 改回字段时，**不要改原文档**——新建 `[Draft] xxx 改版`，链回原 PRD 作为依据。原 PRD 保留作历史档案。
