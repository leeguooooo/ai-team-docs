# ADR Log

每条 ADR 独立追加，编号一旦分配不再回收（包括废弃决策）。

---

## ADR-NNN: {简短标题}

- **日期**：YYYY-MM
- **状态**：已采纳 / 已废弃 / 部分采纳 / 已设计待实施 / 已替代（被 ADR-NNN 替代）
- **背景**：1-3 句话说清楚问题
- **决策**：1-2 句话说清楚选了什么
- **替代方案**：（可选）
  - 方案 A：被否决，原因 ...
  - 方案 B：被否决，原因 ...
- **后果**：
  - 好：...
  - 坏：...
- **重新评估触发条件**：（可选）什么条件下应该回头看这个决策
- **来源**：repo 路径 / 会议 / 讨论链接

---

## ADR-001: 示例决策（可删除）

- **日期**：2026-01
- **状态**：已采纳
- **背景**：单租户表无法支持多 KOL 数据隔离需求
- **决策**：DynamoDB 主键改为复合形式 `kolUserId#resourceId`
- **替代方案**：
  - 每租户独立表：管理成本爆炸
  - 共享表 + FilterExpression：性能差、成本高
- **后果**：
  - 好：天然隔离、避开跨分区扫描、按需计费
  - 坏：主键较长、跨租户查询需要额外 GSI
- **来源**：会议纪要 2026-01-15、PR #123
