# 术语表

> 项目里反复出现、容易踩坑的核心概念。新加入项目或 AI 写文档前先读这页。

## 怎么加新词条

只加满足以下任一条件的词：
- 全公司 / 全项目频繁使用，且新人易混淆
- 同一概念有多个名称（区分内外部命名）
- 历史踩过坑（曾经混用导致 bug）
- 与代码命名严格对应（避免文档与代码漂移）

不加单纯定义型术语（去搜索引擎查更快）。

---

## 核心标识 / ID

| 术语 | 含义 | 用在哪 |
|------|------|--------|
| `内部 ID` | 内部自增/UUID，永不暴露给客户端 | 数据库主键、内部存储 |
| `外部 ID` | 公开短号或 hash，给客户端展示 | API 响应、URL |
| `第三方 ID` | 集成方分配的标识 | 与外部系统通信 |

> ⚠️ **常见踩坑**：把内部 ID 当外部 ID 传给第三方 → 一般会导致查询返空 → 业务侧后果（如轮询误判房间为空、误关闭等）。

## 多租户

| 术语 | 含义 |
|------|------|
| `kolUserId` / `tenantId` | 租户主键 |
| 多租户主键 | 复合主键 `tenantId#resourceId` |
| 多租户切换开关 | 环境变量，平滑切换新旧实现 |
