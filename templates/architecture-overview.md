# 架构总览

## 系统定位

> 1 段话：是什么、不是什么、面向谁。
> 例：本服务是 X 业务的后端单体 HTTP 服务，部署在 Y 平台。它**不是**微服务架构，把 Z 当作底层 provider，自身负责业务规则、多租户数据与运营能力。

## 拓扑（高层）

```
Client (Web/Mobile/Desktop)
    │  HTTPS
    ▼
{LB / Gateway}
    ▼
{App / Service}（部署形态、实例规格）
    │
    ├──► {主存储}
    ├──► {缓存}
    ├──► {外部系统 1}
    ├──► {外部系统 2}
    └──► {可观测 / 告警}
```

要点：
- 路由层来源：手写 / 自动生成（OpenAPI + ogen 之类）
- 单进程 / 多进程：是否同时承载 API + scheduler
- scheduler 是否需要选主
